- Utility Types

Задание: представим, что у нас есть форма регистрации пользователей со следующими полями: 
- email (строка)
- password (строка)
- confirmPassword (строка)
- age (число, необязательное)
- acceptTerms (булево)

Что нужно сделать:
а) Необходимо создать общий интерфейс данных всей формы RegistrationFormData 
б) На его основе создать тип RegistrationFormDraft (промежуточное состояние, когда форма ещё не полностью заполнена => все поля должны быть опциональны)
в) Реализовать тип RegistrationCredentials, содержащий только email и password
г) Реализовать тип FormWithoutConfirmation для формы без подтверждения пароля
